Deploying Streamlit Apps to Streamlit Community Cloud

Yale School of Management
MSOM 630: Gen AI and Social Media
Prof. Zaman

Overview

This guide will walk you through the process of deploying your Streamlit applications to the Streamlit Community Cloud platform. By the end of this tutorial, you'll have your app accessible on the web for anyone to use.

Note: Make sure you have your Streamlit app code ready before starting this process. You should have a functioning app that runs locally using streamlit run app.py.

Step 1: Create a GitHub Account

1.1 Sign Up for GitHub

If you don't already have a GitHub account, you'll need to create one:

  1. Go to https://github.com/
  2. Click on "Sign up" in the top-right corner
  3. Follow the prompts to create your account
    • Enter your email
    • Create a password
    • Choose a username
  4. Verify your email address when prompted

Step 2: Create a New Repository

2.1 Create a New Repository

Once logged into GitHub:

  1. Click the "+" icon in the top-right corner, then select "New repository"
  2. Name your repository (e.g., "streamlit-app" or something more specific to your project)
  3. Add a description (optional but recommended)
  4. Keep the repository Public (required for free Streamlit Community Cloud deployment)
  5. Check the box to "Add a README file"
  6. Click "Create repository"
Tip: Use a clear, descriptive name for your repository. This will help you and others identify your project later.

Step 3: Download and Install Git

3.1 Download Git

Download Git from the official website based on your operating system:

For Windows:

  1. Go to https://git-scm.com/download/win
  2. The download should start automatically
  3. Run the installer and follow the prompts
    • Accept the default settings for most options
    • When asked about "Adjusting your PATH environment", select "Git from the command line and also from 3rd-party software"
    • For "Configuring the line ending conversions", choose "Checkout Windows-style, commit Unix-style line endings"

For Mac:

Option 1: If you have Homebrew installed:

brew install git

Option 2: Download the installer:

  1. Go to https://git-scm.com/download/mac
  2. Download the latest Git for Mac installer
  3. Open the downloaded .dmg file and run the installer

3.2 Verify Git Installation

After installation, verify Git is correctly installed by opening a terminal (Command Prompt on Windows, Terminal on Mac) and running:

git --version

You should see the Git version number displayed.

Step 4: Configure Git

4.1 Set Your Identity

Configure your Git identity with your name and email (use the same email as your GitHub account):

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

4.2 Verify Your Git Configuration

You can check your Git identity settings with these commands:

git config user.name
git config user.email

To view all your Git configuration settings:

git config --list

Step 5: Set Up VS Code

5.1 Download and Install VS Code

If you don't already have VS Code installed:

  1. Go to https://code.visualstudio.com/
  2. Download the appropriate version for your operating system
  3. Run the installer and follow the prompts

5.2 Install Helpful Extensions

VS Code has several extensions that make working with Python and Streamlit easier:

  1. Open VS Code
  2. Click on the Extensions icon in the sidebar (or press Ctrl+Shift+X / Cmd+Shift+X)
  3. Search for and install the following extensions:
    • "Python" (by Microsoft)
    • "Pylance" (by Microsoft)
    • "GitHub Pull Requests and Issues" (by GitHub)
    • "Streamlit" (for Streamlit code snippets and syntax highlighting)

Step 6: Clone Your Repository

6.1 Clone with VS Code

  1. Open VS Code
  2. Press F1 or Ctrl+Shift+P (Cmd+Shift+P on Mac) to open the command palette
  3. Type "Git: Clone" and select that option
  4. Enter the URL of your GitHub repository (e.g., https://github.com/yourusername/streamlit-app.git)
  5. Select a local folder to store your repository
  6. When prompted, open the cloned repository

6.2 Alternative: Clone with Terminal

If you prefer using the terminal:

  1. Open a terminal or command prompt
  2. Navigate to the directory where you want to store your project
  3. Run the following command, replacing the URL with your repository URL: 
    git clone https://github.com/yourusername/streamlit-app.git
  4. Navigate into the cloned repository: 
    cd streamlit-app
  5. Open VS Code in this directory: 
    code .

6.3 Verify Repository Connection

Check that your local repository is correctly connected to GitHub:

git remote -v

This will display your remote repository URLs. You should see something like:

origin  https://github.com/yourusername/streamlit-app.git (fetch)
origin  https://github.com/yourusername/streamlit-app.git (push)

Step 7: Set Up Your Streamlit App Files

7.1 Create App Files

Now that you have your repository cloned locally, add your Streamlit app files to the repository:

  1. Create a new file called app.py (or use your existing file)
  2. Add your Streamlit app code to this file
  3. Create a requirements.txt file listing all the Python packages your app needs: 
    streamlit==1.27.0
pandas==2.0.3
numpy==1.24.3
matplotlib==3.7.2
# Add any other packages your app requires

7.2 File Structure

Your repository should have a structure similar to this:

streamlit-app/
├── README.md
├── app.py
├── requirements.txt
└── data/ (optional folder for any data files)
    └── dataset.csv
Note: If your app uses any data files, make sure to include them in your repository.

Step 8: Create a GitHub Personal Access Token

Before pushing to GitHub for the first time, you'll need to create a Personal Access Token (PAT). GitHub no longer accepts regular passwords for Git operations.

8.1 Create a Personal Access Token

  1. Go to https://github.com/settings/tokens (or navigate to GitHub → Settings → Developer settings → Personal access tokens)
  2. Click "Generate new token" (or "Generate new token (classic)")
  3. Give your token a descriptive name (e.g., "Streamlit App Development")
  4. Set an expiration date (30 days is a good starting point for class projects)
  5. Select scopes/permissions:
    • At minimum, check the "repo" box to get full control of repositories
  6. Scroll down and click "Generate token"
  7. IMPORTANT: Copy your token immediately! GitHub will only show it once.
  8. Store this token somewhere secure (like a password manager) for the duration of this project
Note: You'll use this token in place of your password when Git asks for credentials during your first push.

Step 9: Push Your Code to GitHub

9.1 Stage Your Changes

Using VS Code's Git integration:

  1. Click on the Source Control icon in the sidebar (or press Ctrl+Shift+G / Cmd+Shift+G)
  2. Review the changes listed
  3. Click the "+" (Stage Changes) button next to each file you want to include (or use the "+" button next to "Changes" to stage all)

9.2 Commit Your Changes

  1. Enter a commit message in the text box (e.g., "Add initial Streamlit app code")
  2. Click the checkmark (✓) button to commit the changes

9.3 Push to GitHub

  1. Click the "..." menu in the Source Control panel
  2. Select "Push"
  3. When prompted for authentication:
    • Enter your GitHub username
    • For the password, paste the Personal Access Token you created
  4. After successful authentication, Git will remember your credentials for future pushes

9.4 Alternative: Push with Terminal

If you prefer using the terminal:

git add .
git commit -m "Add initial Streamlit app code"
git push origin main

When prompted, enter your GitHub username and use your Personal Access Token as the password.

Tip: If you're having authentication issues, you can verify your credential helper is working: 
git config --get credential.helper
This should show which credential helper your Git is using to store authentication information.

Step 10: Deploy to Streamlit Community Cloud

10.1 Sign Up for Streamlit Community Cloud

  1. Go to https://streamlit.io/cloud
  2. Click "Sign up" or "Get started"
  3. Sign in with your GitHub account
  4. Complete any required authorization steps

10.2 Deploy Your App

  1. Once logged in, click "New app"
  2. Select your repository, branch (main), and Python file (app.py)
  3. Click "Deploy!"
  4. Wait for the deployment to complete (this may take a few minutes)
Note: Streamlit Community Cloud will automatically detect and install the packages listed in your requirements.txt file.

10.3 Access Your App

Once deployment is complete, your app will be available at a URL like:

https://username-app-name-streamlit-app-xxxxx.streamlit.app/

You'll see this URL on your Streamlit Cloud dashboard.

Step 11: Update Your App

11.1 Make Changes

  1. Edit your app files locally in VS Code
  2. Test your changes by running the app locally: 
    streamlit run app.py

11.2 Push Updates

  1. Stage, commit, and push your changes as in Step 9
  2. Streamlit Community Cloud will automatically detect the changes and rebuild your app
Tip: You can view deployment logs on the Streamlit Community Cloud dashboard to troubleshoot any issues.

Common Issues and Troubleshooting

Issue Solution
Missing dependencies Make sure all required packages are listed in requirements.txt
File not found errors Check file paths - use relative paths in your code
Authentication issues with Git Ensure you've configured Git with the correct credentials. Try refreshing your token, or check your credentials with git config --get credential.helper
Deployment fails Check the logs in Streamlit Cloud for specific error messages
"ModuleNotFoundError" in deployed app Add the missing package to your requirements.txt file
Git says "fatal: not a git repository" Make sure you're in the right directory. Use pwd to check your current location and cd to the repository folder
Stuck in a text editor after a Git command If in Vim: press Escape, then type :q! and hit Enter. If in Nano: press Ctrl+X, then N to exit without saving

Best Practices

  • Environment Management: Consider using virtual environments for local development
  • Version Control: Make small, frequent commits with clear messages
  • Secrets: Never commit API keys or passwords - use Streamlit's secrets management
  • Documentation: Keep your README.md updated with installation and usage instructions
  • Testing: Test your app thoroughly before deployment
Remember: The free tier of Streamlit Community Cloud has some limitations:
  • Apps go to sleep after inactivity
  • Limited compute resources
  • Public repositories only
For more advanced needs, consider Streamlit's paid options or other deployment platforms.
